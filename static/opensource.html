dbt Packages & Data Tools

These packages are used in production analytics workflows to improve documentation, data quality, and observability.

dbt-doc-tracker
★ 6

A dbt package that detects and records documentation changes across your dbt project. Instead of storing full snapshots, it only records the diffs — when a description is added, modified, or removed. Built to support documentation governance and compliance tracking.

Use Case
Documentation governance & audit trails
How It Works
Tracks diffs in model/column descriptions across dbt runs

dbt-doc-inherit
★ 3

A dbt package that propagates column descriptions along the DAG — define once, inherit downstream. Eliminates the repetitive work of documenting the same column across dozens of models. Solves the "documentation drift" problem at scale.

Use Case
DRY documentation across dbt DAGs
How It Works
Propagates descriptions from parent to child models automatically

dbt-autodoc

Auto-generate dbt model and column descriptions using heuristics or LLMs. This is the tool behind the 35%+ documentation coverage improvement at The Bredge. Bridges the gap between "we should document everything" and actually doing it.
